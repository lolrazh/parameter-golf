### SWA Implementation
```python
# Collect snapshots during last swa_start_frac of warmdown
if args.swa_enabled and scale < 1.0:
    warmdown_progress = 1.0 - scale
    if warmdown_progress >= (1.0 - args.swa_start_frac) and not swa_collecting:
        swa_collecting = True
    if swa_collecting and step % args.swa_every == 0:
        swa_snapshots.append({k: v.detach().cpu().float().clone() ...})

# Average at end
avg_sd = {key: torch.stack([s[key] for s in swa_snapshots]).mean(dim=0) ...}
base_model.load_state_dict(avg_sd, strict=True)
```

## Bugs & Issues Encountered
1. **Spot instance preemption (2x)** — Both 8xH100 spot pods got reclaimed mid-run. Even "on-demand" had issues (second pod died during setup, cause unclear).
   - **Workaround:** Plan to use Thunder Compute (1xH100 PCIe) for iteration, save 8xH100 for final validated runs only.
2. **FA3 slower than SDPA on 1xH100** — 182ms/step (FA3) vs 132ms/step (SDPA) on 1xH100 SXM with 131K batch. FA3 needs larger batches to shine.
   - **Fix:** Use SDPA for 1xH100 dev runs. FA3 only helps on 8xH100 (59ms/step with 786K batch).
3. **10L SWA + warmdown=3000 hurt** — 10L sp4096 with SWA+warmdown=3000 got 1.1587 vs 9L's 1.1484 without those tricks. Too much time at low LR.
   - **Learning:** SWA and long warmdown need more total steps to help. At ~7000 steps, warmdown=1200 is better.
4. **9L sp4096 artifact too large** — 17.3 MB with int6+zlib, 1.3 MB over 16 MB limit. Need int5 MLP + zstd to fit.
   - **Fix:** INT5_MLP_ENABLED=1 saves ~1.86 MB, zstd-22 compresses better than zlib-9. 10L version fit at 15.8 MB.

## Key Learnings
- **8xH100 SXM is 12-14x faster than 1xH100 PCIe in wall-clock steps.** 6L: 10,200 steps/10min (8x) vs 4,537 (1x). DDP + NVLink + smaller per-GPU batch = massive throughput.
- **FA3 helps at scale, hurts at small batch.** FA3 on 1xH100 (131K batch): 182ms. SDPA: 132ms. FA3 on 8xH100 (786K batch): 59ms per step. The Hopper-specific async warp groups need saturated SMs.
- **9L is the sweet spot for sp4096.** 1.1484 BPB, but needs int5+zstd to fit 16 MB. 10L adds a layer but not enough steps to use it (7,131 vs 7,249 for 9L).
- **SWA and long warmdown need more steps.** SOTA uses warmdown=3000 with ~8,945 steps (33%). Our 10L got 7,131 steps — warmdown=3000 means 42% at low LR, too aggressive. Stick with warmdown=1200 at our step counts.
- **Int5 MLP is essentially free quality-wise.** SOTA ablation shows int5 MLP → int6 attention with an extra layer beats uniform int6 with fewer layers. The MLP weights are most compressible.
- **Spot 8xH100 is unreliable.** Two preemptions in one session. For final submission runs, use on-demand or have a retry strategy.
- **Setup from scratch takes <3 min.** Clone + FA3 wheel (6s) + zstd + data download (~50s) = under 3 minutes. No network volume needed.

## Architecture Decisions
- **Our LR/momentum over SOTA's** — Our tuning (matrix_lr=0.04, momentum=0.95) was validated on proxy runs. SOTA uses 0.02/0.99. We chose not to copy blindly since our optimizer (batched Muon) and architecture (fused QKV) differ.
- **No SWA for submission runs** — SWA hurt on 10L (1.1587 vs 9L's 1.1484 without). Will revisit if we get more steps on better hardware.
- **Save exact run snapshots** — Each 8xH100 run gets its own `runs/<config>/train_gpt.py` + `run.sh` for PR reproducibility.
- **Thunder Compute for iteration** — Use 1xH100 PCIe ($2.49/hr) with known 8xH100 baselines to predict whether a config change would beat SOTA. Only use 8xH100 for final validated runs.
