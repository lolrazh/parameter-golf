Fix is trivial but not yet applied.
- ⚠️ **10-min baseline launched** — Started with FA3 enabled but crashed on the reshape bug. Needs the `[0]` fix, then re-run.

## Bugs & Issues Encountered
1. **MUON_USE_CUDA_GRAPH=1 default causing NaN** — Every training run got NaN at step 3. The CUDA graph capture path has a double-update bug: graph capture EXECUTES the step, then the optimizer also runs `_batched_step`, causing two weight updates on step 2 and corrupted state from step 3.
   - **Fix:** Changed default from `"1"` to `"0"`. Batched Muon alone gives 4.8x speedup; CUDA graph adds only ~1.3ms more.
2. **FA3 from-source compilation: 451 .cu files, ~12+ hours** — Tried `pip install -e .` in flash-attention/hopper/. Each kernel instantiation takes 2-3 min. `MAX_JOBS=32` didn't help (setuptools compiled one file at a time despite the flag).
   - **Fix:** Pre-built wheels from windreamer. 6 seconds.
3. **FA3 reshape crash** — `flash_attn_3_func` output indexed with `[0]` strips batch dimension. Error: `shape '[8, 2048, 512]' is invalid for input of size 1048576`.
   - **Fix (pending):** Remove `[0]` or check if return type changed between FA3 versions. PR #198 doesn't use `[0]`.
4. **PEP 668 pip errors on RunPod** — Default container blocks pip installs.
   - **Fix:** `pip install --break-system-packages`
5. **SSH exit code 255 on multi-command strings** — Complex chained commands over SSH fail intermittently.
   - **Workaround:** Write bash scripts, SCP them to pod, run via `nohup`.
6. **sp4096 data not in official manifest** — `cached_challenge_fineweb.py --variant sp4096` fails. Data is on sproos/parameter-golf-tokenizers HuggingFace repo, not the official download script.
   - **Workaround:** Download directly via `huggingface_hub.hf_hub_download`.

## Key Learnings
- **FA3 pre-built wheels exist and install in 6 seconds.** The `flash_attn_3` package from windreamer has wheels for all major PyTorch+CUDA combos. Never compile from source. Index: `https://windreamer.github.io/flash-attention3-wheels/`
- **PR #290 scored 1.1354 BPB WITHOUT FA3.** SDPA fallback works fine for competitive results. FA3 gives ~600 more training steps (~7% throughput gain) but is not blocking.
- **Network volumes are datacenter-locked on RunPod.** Must create the volume in the same DC as the GPU pod. Not all DCs support volume creation (US-CA-2 refused).
- **The official RunPod Parameter Golf template does NOT include FA3.** Issue #280 confirms this. The eval environment may differ from the template.
- **Spot H100 SXM at $1.75/hr makes the 4090 setup trick unnecessary.** Setup cost on H100 SXM is ~$0.58 (20 min). The $0.40 savings from a 4090 isn't worth the cross-architecture compilation issues (FA3 kernels are SM90-specific).
- **MUON_USE_CUDA_GRAPH should default to OFF.** The CUDA graph capture path has a known double-update bug that causes NaN. Batched Muon alone is sufficient (4.8x speedup).

## Architecture Decisions
- **Use official Parameter Golf template over custom** — Template has PyTorch 2.9.1, CUDA 12.8, all base deps. No need to maintain our own Docker image. Code goes on the volume or is cloned fresh.
- **SDPA fallback preserved in sota_train_gpt.py** — Keep `try/except` for FA3 import during development. Switch to bare import only in final submission `train_gpt.py`.
- **SCP scripts instead of inline SSH commands** — Complex multi-command SSH strings fail with exit 255. Writing .sh scripts and SCP'ing them is more reliable.
